Git团队协作机制
	团队内协作：代码托管中心 
	push推送到托管中心远程库（GitHub），拿代码的人克隆下来clone，修改后再push（  需要权限），可以拉取pull下来，更新自己的本地库
	跨团队协作：A远程库 fork叉过来（复制到） B远程库，B远程库可以clone，本地库中修改后push到B远程库，要传递给A远程库需要发起pull request拉取请求，A远程库审核，审核后可以merge合并到自己的A远程库，A远程库的成员可可以pull了。
	（Git版本控制都是在本地库中做的，所以称git为分布式版本控制工具）

创建远程仓库别名
	git remote -v 查看当前所有远程地址别名 别名既可以拉取也可以推送所以是两个
	git remote add 别名 远程地址
	
推送本地库到远程库
	git push 别名 分支 （最小单位是分支，需要写分支名）

远程库拉取到本地库
	git pull 别名 分支 （最小单位是分支，需要写分支名）
